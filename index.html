Will's AI Workflow

Mindvalley Organic Social — How I use Claude across my day

Phase 1 — Inputs
📥
Gather Raw Inputs
Everything starts here
I feed in whatever I'm working with — Slack threads, Airtable data, video transcripts, uploaded docs (PRDs, strategy docs, brand guidelines), CSV exports from Metabase/Looker, messaging documents, or just a raw idea or question. Claude pulls additional context from connected MCPs (Slack, Airtable, Google Drive, Figma) and runs web research when needed.
Slack MCP Airtable MCP Google Drive Figma File uploads Web search
Phase 2 — Analysis & Synthesis
🔍
Research & Analyse
Make sense of the raw material
Claude cross-references inputs against trending news, platform best practices, competitor strategies, and internal data (Airtable project tracker, content calendars, team workloads). It identifies what matters — whether that's a data error in a leads dashboard, a gap in the content calendar, or the right angle for today's LinkedIn post.
Trend scanning Data validation Competitive intel Content calendar mapping UTM/attribution debugging
Phase 3 — Production
✍️
Daily LinkedIn Content Daily
3 posts for Vishen's profile + 1 for the company page. Each with full copy, format (text / carousel / video brief), comment-trigger CTAs for ManyChat routing, and publish timing. Carousel posts include slide-by-slide design outlines.
Custom skill Markdown briefs
📋
Strategy & Research As needed
Deep dives that produce comprehensive documents — YouTube strategy, team priorities, TikTok Shop playbooks, channel maps, campaign task lists. These become the source of truth that other outputs flow from.
Markdown .docx .xlsx Presentations
📊
Data & Operations Weekly
Task management via Airtable — pulling team tasks, creating records from Slack, building RACI matrices. Data validation on campaign metrics (leads, attribution, UTMs). Content calendar exports and summaries.
Airtable CRUD CSV analysis Slack extraction
💬
Comms & Documents As needed
Drafting messages in my voice (Slack style saved in memory), offboarding letters, vendor correspondence, editor briefs from video transcripts, YouTube descriptions. Quick-turn outputs that go straight to people.
Writing style memory .docx letters Paper edits
Phase 4 — Build Interactive Tools
⚙️
Dashboards & Command Centres Project
Turn strategy into something the team can use
Strategy docs and data get turned into interactive React artifacts or HTML tools — dashboards with live checklists, command centres with embedded intelligence, ROI calculators, editable role profiles, architecture maps. Heavy iteration on UI: tightening spacing, adding edit modes, fixing rendering for Claude artifact sandbox. These are designed for the team to open and use directly.
React / JSX artifacts HTML dashboards Figma (with fallback) Google Stitch prompts Tailwind + shadcn/ui
Phase 5 — Iterate Until Team-Ready
🔄
Refine & Polish
Most sessions don't end on the first pass
I push back on outputs until they're right — "still too tall", "this isn't a flowchart", "I want it to actually generate, not produce a prompt". Claude rebuilds, compresses, restructures. Typically 2–4 rounds. The goal is always: can someone on my team open this and use it without asking me questions?
UI tightening Format changes Edit mode / export Cross-platform testing
Phase 6 — Deliver
🚀
Share With Team & Execute
Files, artifacts, Airtable records, messages
Finished outputs go to the team as downloadable files (.docx, .xlsx, .html, .jsx), Claude artifacts they can open in their own sessions, Airtable records pushed directly into the project tracker, Slack messages drafted in my voice, or Figma files. Daily LinkedIn briefs go straight into the publishing pipeline.
Team downloads Claude artifacts Airtable push Slack drafts Figma Publishing pipeline
The Meta Pattern
Raw inputs → Analysis & synthesis → Strategy or content → Interactive tool → Iterate until team-ready → Ship.

I'm using Claude as a one-person ops layer that sits between my data sources (Slack, Airtable, Metabase, Dropbox, transcripts) and my team's execution. It turns fragmented information into polished, usable outputs — daily content, strategy docs, dashboards, and operational records — so the team can act without asking me to decode everything first.
